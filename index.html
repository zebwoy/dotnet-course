1.Class

A class is an abstract blueprint used to create more specific objects.

2.Object

An object is an instance derived from a class.

Example: A class can be viewed like the cookie cutter. Every cookie (derived object) from this blueprint will have the same basic structure. But very cookie (object) could be further individualized with colored icing, toppings, etc…

3.Abstraction

Main goal is to Hide complexity details from the user and use simple things to represent complexity

4.Encapsulation

Pack all important information into a single package using curly braces { }

Polymorphism

Polymorphism is mainly divided into two types:

  • Compile-time Polymorphism
